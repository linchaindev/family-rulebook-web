1. 프로젝트 개요

제작 배경

KH 패밀리룰북은 가족 구성원 5인(부모 2인, 자녀 3인)이 함께 합의한 생활 규칙을 디지털로 관리하기 위해 2026년 2월에 제작되었습니다. 종이 규칙집이나 메모 앱으로는 추적이 어려웠던 스크린타임 기록, 용돈 정산, 매니저 평가 등을 하나의 웹앱으로 통합했습니다.

단순한 규칙 열람을 넘어, 실제 데이터를 기록하고 월말에 자동으로 정산하는 운영 가능한 가족 거버넌스 시스템을 목표로 설계했습니다.

핵심 목표

투명성

모든 기록(스크린타임, 카드, 용돈)이 DB에 저장되어 누구나 조회 가능

자동화

월말평가 완료 시 DDC 순위 → 상금/벌금 → 용돈 정산까지 자동 계산

접근 제어

FA(패밀리 감사) 6자리, FM(패밀리 매니저) 4자리 비밀번호로 권한 분리

확장성

가족 구성원 수, 규칙 내용, 상금 체계를 설정 파일 수정만으로 변경 가능

3. 시스템 아키텍처

데이터 흐름

Browser (React)
  │  fetch /api/trpc/* (httpBatchLink + superjson)
  ▼
Express Server
  │  server/_core/context.ts → ctx.user 주입
  ├─ publicProcedure  (로그인 불필요)
  └─ protectedProcedure (JWT 세션 쿠키 검증)
       │
       ▼
  server/routers.ts (tRPC 라우터)
       │  Drizzle ORM
       ▼
  TiDB / MySQL

모든 API 통신은 /api/trpc 단일 엔드포인트를 통해 이루어집니다. tRPC의 배치 링크 덕분에 여러 쿼리가 하나의 HTTP 요청으로 묶입니다.

인증 구조

Manus OAuth 흐름 로그인 → /api/oauth/callback → JWT 세션 쿠키 발급 → 이후 모든 요청에 쿠키 자동 첨부

이 프로젝트는 Manus 플랫폼의 OAuth를 사용하므로, fork 후 다른 OAuth 제공자(Google, Kakao 등)로 교체하려면 server/_core/oauth.ts와 server/_core/context.ts를 수정해야 합니다.

앱 내부에는 추가로 두 가지 비밀번호 레이어가 있습니다:

역할자릿수저장 위치용도
FM (패밀리 매니저)4자리passwords 테이블 (월별)매니저 입력 페이지 접근
FA (패밀리 감사)6자리auditor_config 테이블감사 관리 페이지 접근

4. 데이터베이스

스키마 설계 원칙

  • 날짜는 VARCHAR(10)의 YYYY-MM-DD 형식으로 저장 (타임존 혼란 방지)
  • 월(month)은 VARCHAR(7)의 YYYY-MM 형식
  • 금액은 만원 단위 정수로 저장 (소수점 불필요)
  • 모든 테이블에 createdAt, updatedAt 타임스탬프 포함
  • 중복 방지가 필요한 경우 unique() 제약 조건 사용

5. 핵심 기능

DDC — 디지털 디톡스 챌린지

가족 구성원별 일일 스크린타임을 기록하고, 월간 총합을 기준으로 순위를 매겨 상금/벌금을 산정합니다.

순위상금/벌금
1등+5만원
2등+3만원
3등0원
4등-3만원
5등-5만원
구현 포인트 ddc_records에 unique(date, member_id) 제약이 있어 같은 날 같은 멤버의 데이터는 INSERT ... ON DUPLICATE KEY UPDATE로 자동 업데이트됩니다.

RCR — 규칙 준수 카드 시스템

10가지 카드 타입으로 규칙 위반/준수를 기록합니다. 카드 효과는 스크린타임 조정 또는 용돈 조정으로 나뉩니다.

yellow: 스크린타임 +5h red: 용돈 -1만 double_red: -2만 triple_red: -3만 quadro_red: -4만 green: 스크린타임 -1h double_green: -5h triple_green: 용돈 +2만 quadro_green: +4만 golden: 매니저 1개월 면제

용돈 정산 시스템

월말평가 완료 시 아래 공식으로 최종 용돈이 자동 계산됩니다:

최종 용돈 = 기본 용돈
         + DDC 상금/벌금
         + RCR 카드 효과 합산
         + FA 버프/너프 (allowance_adjustments)
         + 매니저 보상 (해당 월 매니저인 경우)

계산 결과는 monthly_allowances.breakdown_formula에 문자열로 저장되어 프로필 페이지에서 항목별로 표시됩니다.

월말평가 흐름

1

투표

가족 5인이 당월 매니저에 대해 good/bad 투표 (자기 자신 제외 4표)

2

집계

good 비율에 따라 매니저 보상 결정 (80% 이상 → 최대 보상)

3

정산

DDC 순위 스냅샷 저장 → 5인 전체 용돈 자동 계산 → monthly_allowances 저장

4

다음 달 준비

새 FM 비밀번호 자동 생성 → 오너에게 알림 전송

패밀리 감사(FA) 관리 페이지

/auditor-admin 페이지는 6자리 FA 비밀번호 인증 후 접근 가능합니다. 4개 탭으로 구성됩니다:

  • DDC 탭 — 스크린타임 데이터 입력/수정
  • RCR 탭 — 카드 발급/취소
  • 용돈/버프 탭 — 버프·너프 추가, 기본 용돈 설정
  • 평가 관리 탭 — 월말평가 진행/취소(롤백)

6. Fork 가이드

이 프로젝트를 기반으로 새로운 패밀리룰북을 만들려면 아래 단계를 따르세요.

환경 설정

1

저장소 Fork

GitHub에서 이 저장소를 Fork한 뒤 로컬에 클론합니다.

2

의존성 설치

pnpm install 실행. Node.js 22+, pnpm 9+ 필요.

3

환경 변수 설정

아래 환경 변수를 .env 파일 또는 호스팅 플랫폼에 등록합니다.

4

DB 마이그레이션

pnpm db:push로 스키마를 DB에 적용합니다.

5

개발 서버 실행

pnpm dev로 localhost:3000에서 확인합니다.

필수 환경 변수

DATABASE_URL=mysql://user:password@host:4000/dbname?ssl={"rejectUnauthorized":true}
JWT_SECRET=your-random-32-char-secret
VITE_APP_ID=your-manus-app-id
OAUTH_SERVER_URL=https://api.manus.im
VITE_OAUTH_PORTAL_URL=https://manus.im
OWNER_OPEN_ID=your-manus-open-id
OWNER_NAME=your-name
BUILT_IN_FORGE_API_URL=https://api.manus.im
BUILT_IN_FORGE_API_KEY=your-api-key
주의 Manus OAuth를 사용하지 않는 경우 server/_core/oauth.ts, server/_core/context.ts, client/src/const.ts의 인증 로직을 교체해야 합니다.

server/routers.ts의 DDC_PRIZES 객체를 수정합니다:

  3: 0,   // 3등: 0원
  4: -3,  // 4등: -3만원

};

기본 용돈 변경

FA 관리 페이지 → 용돈/버프 탭에서 멤버별 기본 용돈을 직접 설정할 수 있습니다. 코드 수정 없이 UI에서 변경 가능합니다.

RCR 카드 타입 추가

drizzle/schema.ts의 card_type enum에 새 타입을 추가하고 pnpm db:push를 실행합니다. 이후 server/routers.ts의 카드 효과 계산 로직에 새 타입의 효과를 추가합니다.

앱 이름/로고 변경

# Manus 플랫폼 환경 변수
VITE_APP_TITLE=우리 가족 룰북
VITE_APP_LOGO=https://your-logo-url.png

배포

Manus 플랫폼을 사용하는 경우 GitHub 저장소를 연결하면 main 브랜치 푸시 시 자동 배포됩니다. 다른 플랫폼을 사용하는 경우:

# 빌드
pnpm build

# 빌드 결과물
dist/          ← 프론트엔드 정적 파일
dist/index.js  ← Express 서버 번들

# 실행
node dist/index.js
포트 설정 서버는 process.env.PORT를 읽습니다. 하드코딩하지 마세요. Railway, Render, Fly.io 등 대부분의 플랫폼과 호환됩니다.

8. API 레퍼런스

모든 API는 tRPC 프로시저로 구현되어 있습니다. 클라이언트에서 trpc.<namespace>.<procedure>.useQuery() 또는 .useMutation()으로 호출합니다.

DDC 관련

프로시저타입설명
ddc.getRecordsQuery월별 스크린타임 기록 조회
ddc.upsertRecordsMutation스크린타임 일괄 저장/수정
ddc.getMonthlyRankingQuery월별 DDC 순위 계산
ddc.getMonthlyStatsQuery월별 통계 (총합, 평균)

RCR 관련

프로시저타입설명
rcr.getRecordsQuery카드 기록 조회
rcr.createRecordMutation카드 발급
rcr.deleteRecordMutation카드 취소

월말평가 관련

프로시저타입설명
managerEvaluation.voteMutation투표 제출
managerEvaluation.getVotesQuery투표 현황 조회
managerEvaluation.completeEvaluationMutation평가 완료 + 용돈 정산
managerEvaluation.cancelAndRollbackMutation평가 취소 + 데이터 롤백

용돈 관련

프로시저타입설명
allowance.getMonthlyAllowancesQuery월별 용돈 정산 결과 조회
allowance.getAdjustmentsQuery버프/너프 내역 조회
allowance.addAdjustmentMutation버프/너프 추가
allowance.deleteAdjustmentMutation버프/너프 삭제
